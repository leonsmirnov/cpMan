App Store Review Notes — cpMan (macOS)
========================================
Paste the sections below into App Store Connect → App Review Information → Notes (or attach this file). Adjust dates/names if needed.

────────────────────────────────────────
1. What this app is
────────────────────────────────────────
cpMan is a menu bar clipboard manager for macOS. It records text and images you copy, keeps a searchable history, and lets you paste any previous item quickly via a floating picker and keyboard shortcuts.

The app uses LSUIElement: it does NOT show a Dock icon or ⌘Tab entry by design. Access is from the menu bar (clipboard icon) and the global hotkey.

What's new in 2.0: image clips with on-device OCR (Vision), optional auto-paste, Private Mode (pause recording), an ignore list (skip chosen apps), and configurable history limits.

────────────────────────────────────────
1a. Demo content for review (recommended — see real data immediately)
────────────────────────────────────────
A fresh clipboard history is empty until you copy something. To review the UI immediately, launch cpMan in DEMO MODE, which pre-loads fictional sample clips (text + two sample images with OCR text). No real user data is used.

IMPORTANT: cpMan is a menu bar agent. If it is already running, passing launch arguments does nothing — you must quit it first and start a NEW process. Run this in Terminal:

  killall cpMan 2>/dev/null; open -n -a "/Applications/cpMan.app" --args -CPManDemoMode

Then press ⌃⌥V to open the picker — you'll see ~12 text clips and 2 image clips. Type in the search field (e.g. "roadmap" or "git") to test filtering. Select an image clip and choose "Paste as Plain Text" to paste its OCR-extracted text.

Demo mode is NOT reachable from the normal UI; it only activates via this launch flag, so regular users never see seeded content.

────────────────────────────────────────
2. How to launch and find the UI (critical)
────────────────────────────────────────
After launch, look for the clipboard-style icon in the menu bar (top right area).

Open the history picker with the default global shortcut:
  Control + Option + V   (⌃⌥V)

(Users can change this in cpMan → Settings → Hotkeys.)

If the menu bar is crowded, click the chevron (») on the menu bar to reveal hidden icons, or quit other menu bar apps temporarily.

Open Settings via:
  Menu bar icon → Settings…
or standard macOS Settings if cpMan exposes a Settings scene (menu bar app).

────────────────────────────────────────
3. Accessibility permission (auto-paste)
────────────────────────────────────────
Optional feature: “Auto-paste on selection” pastes into the previously active app using a synthesized ⌘V.

macOS requires Accessibility approval for synthetic key events.

First-time flow:
  • Open the picker (⌃⌥V). If Accessibility is not granted and auto-paste is enabled, an orange banner appears at the top of the picker with “Open Settings”.
  • That deep-links toward System Settings → Privacy & Security → Accessibility.
  • Enable cpMan in the list (user may need to add the app with +, picking /Applications/cpMan.app).

Without Accessibility, cpMan still works fully: selecting an item copies it to the clipboard and the user presses ⌘V manually. Auto-paste can be turned OFF in Settings → General, in which case the app never requests Accessibility at all.

Scope of Accessibility use (important): cpMan uses Accessibility ONLY to synthesize a single ⌘V keystroke into the previously active app after the user explicitly selects an item. It does NOT read, log, or monitor keystrokes from other apps, and it respects Secure Event Input (it will not act while a secure/password field is focused).

There is no login, no server, and no account system.

────────────────────────────────────────
4. Data / privacy / sandbox (technical)
────────────────────────────────────────
• Clipboard contents and history stay on device.
• History is stored under the app container (Application Support) when running sandboxed.
• No analytics SDK is bundled; privacy nutrition labels should reflect “Data Not Collected” unless you add networking later.

5. Suggested review checklist
────────────────────────────────────────
□ Launch in demo mode (see section 1a) → menu bar icon visible
□ Press ⌃⌥V → picker appears with sample text + image clips
□ Type in search field → list filters
□ Copy text in Safari/Notes → appears at top of list
□ Select an image clip → "Paste as Plain Text" pastes its OCR text
